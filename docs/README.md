# Документация для Защиты Диплома

Этот набор файлов нужен не разработчику, а команде, которая будет защищать проект перед комиссией.

Читайте в таком порядке:

1. `01-project-overview.md` — что это за проект, зачем он нужен, как он работает, почему выбран именно такой подход.
2. `02-system-and-buttons.md` — что делает каждая страница, кнопка, вкладка и основная функция системы.
3. `03-commission-audit.md` — что может спросить комиссия, как лучше отвечать и где нужно отвечать особенно честно.
4. `04-defense-script.md` — готовый человеческий сценарий устного объяснения и демонстрации.

Главное правило защиты:

- Не пытайтесь выглядеть "слишком умными".
- Говорите просто.
- Не врите там, где в проекте есть демо-логика или ограничения.
- Если не знаете глубокой теории, объясняйте через идею: "сначала ищем странное, потом называем тип атаки".

Коротко о проекте:

- Название: `NetShield IDS`
- Тип проекта: учебная система обнаружения сетевых атак
- Формат: веб-приложение с фронтендом, бэкендом, ML-моделями и демонстрационным real-time режимом
- Главная цель: показать, как можно обнаруживать сетевые угрозы с помощью машинного обучения и удобно объяснять результат пользователю
